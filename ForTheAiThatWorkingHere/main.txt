## 架构详解

### 核心架构模式

**分层信标记忆系统** — 三级压缩层级，通过知识图谱路由查询。

```
B1 (1000 tokens)  →  B2 (10 B1s)  →  B3 (10 B2s)
  切线向量              高斯补丁         谱签名
  128-d float16        128-d mu + Sigma  5 eigenvalues + 5 vectors
```

### 数据流

#### 摄入管道（写入路径）

```
原始文本令牌 → 分词器（空格分割）→ 页面切片器（1000令牌/页）
    ↓
B1编码器: sinusoidal_encode() → 切线向量 v_i ∈ ℝ^512
    ↓
实体提取器: HeuristicExtractor → 三元组 (S,P,O) 带 Beacon_ID, Page_ID
    ↓
知识图谱: add_triplet() → 稀疏邻接矩阵 A (CSR格式)
    ↓
B2压缩器: compute_gaussian_patch() → 高斯补丁 G_j = (μ_j, Σ_inv_j)
    |   （每10个B1信标）
    ↓
B3压缩器: spectral_signature() → 特征值 λ(L_sym)
    |   （每10个B2信标）
    ↓
光谱记忆流形 M（统一：向量 + 图 + 时间 + 压缩）
```

#### 查询管道（读取路径）

```
查询 Q → 嵌入 Q → LSH桶 → 概念扩散 → B3排名 → B2扩展 → B1扩展 → 子模块包装 → CollapseResult
```

### 组件图

| 组件 | 文件 | 职责 |
|------|------|------|

### 公式快速参考

**嵌入：**
```
z(x)_j = cos(w_j * MD5(x) + b_j)
w_j ~ N(0, 4.0), seed=42
b_j ~ U(0, 2π), seed=43
```

**LSH哈希：**
```
h_i(x) = floor((a_i · x + b_i) / w)
bucket = (h_1(x), ..., h_m(x))
```

**图拉普拉斯：**
```
L_sym = I - D^(-1/2) * A * D^(-1/2)
```

**Fokker-Planck扩散：**
```
ρ_{t+1} = ρ_t + α·(L·ρ_t) + β·(q ⊙ ρ_t) - γ·ρ_t
α=0.1, β=0.5, γ=0.01
```

**子模块包装：**
```
贪婪算法: m* = argmax(marginal_gain(m) / |m|)
保证: f(S_greedy) >= (1 - 1/e) * f(S_optimal) ≈ 0.63 * OPT
```

**高斯补丁（B2）：**
```
μ = mean(vectors)
Σ = (X-μ)^T * (X-μ) / (n-1)
Σ_inv = pinv(Σ)
```

**置信度分数：**
```
confidence = min(1.0, total_coverage / max(tokens_used, 1) * 100.0)
```

**压缩比：**
```
compression_ratio = tokens_total / tokens_used
```

### 内存布局（每组件）

```
B1嵌入 (float16):     512 × 2 bytes = 1 KB/页
B2补丁 (float64对角):  512 × 8 bytes = 4 KB/补丁（仅对角线）
B3签名 (float64):     5 × 8 bytes = 40 bytes/签名
图邻接 (CSR):         ~8 bytes/边
LSH哈希向量 (float64): 512 × 8 = 4 KB（固定，单一实例）
```

## 已知问题和修复

### 当前关键问题

#### 1. 存储开销问题（严重）
- **问题**：系统存储原始文本 + 嵌入 = 比原始文本大1.3倍
- **当前**：1.09 bytes/token vs 原始文本0.52 bytes/token
- **目标**：< 0.5 bytes/token（比原始文本小）
- **原因**：
  - `self.pages` OrderedDict存储所有文本在RAM中
  - B1嵌入512维 × 2 bytes = 1KB/页
  - B2存储Sigma_inv_diag（4KB/批）
- **修复计划**：
  - 不在RAM存储原始文本（仅磁盘）
  - 减少B1维度512→128
  - 不存储Sigma_inv_diag

#### 2. Multi-needle检索准确率（严重）
- **问题**：60%（目标≥95%）
- **根因**：通用查询"What are the facts?"仅激活1个概念（`['has']`），信标排名无区别
- **修复计划**：
  - 自适应概念激活：当activated_concepts < 3时，激活所有概念
  - 增加B3信标数量5→15

#### 3. Adversarial检索准确率（严重）
- **问题**：20% clean（目标≥90%）
- **根因**：所有查询为正确和干扰页面激活相同概念
- **修复计划**：
  - 改进实体提取器（扩展动词列表）
  - 页面重要性回退（TF-IDF风格）
